# Atlas Workspace / NovaForge Spec Rollup Pack

This pack is a distilled repo-drop audit/spec package built from the project planning and implementation chats available in this workspace.

## Intended use
Drop this pack into a repo and audit the codebase against:
- locked project direction
- AtlasUI standards
- workspace/tooling priorities
- architectural constraints
- feature expectations
- roadmap ordering

## Recommended placement
- `Docs/SpecRollup/`
- or `Audit/SpecRollup/`

## What this pack is
- a practical audit baseline
- a feature and architecture reference
- a roadmap and standards lock
- a repo-gap detection guide

## What this pack is not
- not a raw transcript export
- not a line-by-line code audit of your repo
- not a generated implementation of every system

## Core locked direction
- Atlas Workspace is the main development environment
- NovaForge is developed inside the workspace as a hosted project
- AtlasUI is the mandatory tooling UI standard
- Win32 is the baseline platform direction for initial tooling/apps
- tooling and framework work comes before game feature work by default

See the other markdown files in this pack for the full working standard.
