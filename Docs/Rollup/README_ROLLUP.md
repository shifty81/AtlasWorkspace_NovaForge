# Atlas Workspace + NovaForge Rollup Pack

## Purpose
This pack is a repo-drop rollup built from the current project chat direction, the uploaded `spec.txt`, and the uploaded `tempnovaforge.zip` source snapshot.

## Locked project direction
- Atlas Workspace is the main development environment.
- NovaForge is the main hosted project developed inside that environment.
- Workspace and hosted project stay logically separate even when living inside the same working monorepo.
- Win32 is the baseline platform direction.
- AtlasUI is the standard UI framework.
- Voxel truth layer + low-poly visual wrapper is the locked content/render direction.
- Game development work is pushed behind AtlasUI/framework/tooling standardization unless explicitly chosen.

## Current design spine
- Buried bunker start.
- Shaft excavation to reach surface.
- Freight elevator restoration.
- Centrifuge-based early material economy.
- Backpack-first R.I.G. progression.
- Helmet-required HUD.
- Palm/glove interface port for advanced interactions.
- Embedded R.I.G. AI for alerts, scanning, mapping, drones, and later nano-healing.
- Vehicles, stations, and hacking all route through the same interface philosophy.

## Biggest current blockers
1. Missing canonical contracts and schemas.
2. Missing simulation math for power, oxygen, collapse, and resource flow.
3. Missing state machines for elevator, link, breach, AI feature activation, and healing.
4. Missing progression tables and balancing thresholds.
